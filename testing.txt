TRIAL 9 - Measured-text-width arc span (new)
- Change: compute actual pixel width of the full label using `dc.getTextWidth(roundedFontSmall, text)` and convert that pixel width to an angular span at radius `r`.
- Formula: `spanDeg = (pxWidth * 180.0) / (Math.PI * r)` — then apply a padding factor and clamp.
- Parameters used for Trial 9 (please test these):
	- `paddingFactor` = 1.10 (10% extra so chars don't touch)
	- `minSpanDeg` = 8.0   (degrees — avoid too-narrow arcs)
	- `maxSpanDeg` = 80.0  (degrees — avoid excessive wrapping)
	- Computation steps used:
		1) `px = dc.getTextWidth(roundedFontSmall, text)`
		2) `rawDeg = (px * 180.0) / (Math.PI * r)`
		3) `estSpan = clamp(rawDeg * paddingFactor, minSpanDeg, maxSpanDeg)`
		4) If `text.length() <= 1` then `estSpan = minSpanDeg` (single-char special case)

Sample expected spans (approx) at `r = 119.6` px:
	- "HDG 090" (7 chars): px≈56 → rawDeg≈ (56*180)/(π*119.6) ≈ 26.9° → estSpan ≈ 29.6°
	- "GS 120 kt" (9 chars): px≈72 → rawDeg≈34.7° → estSpan ≈ 38.2°
	- "ALT 12345" (10 chars): px≈80 → rawDeg≈38.6° → estSpan ≈ 42.5°
